📱 CLIENT SIDE
🌐
Browser SDK
JavaScript library embedded in the web app. Silently captures behavioral events (mouse, keyboard, touch) at 100ms intervals using Browser APIs.
💡 "SDK" = Software Development Kit — a pre-built library developers plug into their app
JavaScript 100ms polling TLS encrypted
📱
Mobile Agent
Native SDK for Android & iOS. Captures touch gestures, gyroscope data, device accelerometer, and screen interaction patterns in real time.
Android SDK iOS SDK Gyroscope
📦
Event Batcher
Groups raw events into 100ms micro-batches before sending to the server. Reduces network overhead dramatically while preserving timing precision.
💡 Batching = grouping many small events into one package to save bandwidth
100ms batches gzip compressed
⚡ Event Streaming Layer
Batched events published to Apache Kafka topic queue. Enables async, decoupled, high-throughput streaming between client and server — no data loss under load.
Apache Kafka RabbitMQ alt. Async processing
🖥️ SERVER SIDE
⚖️
Load Balancer
Distributes incoming traffic across multiple API Gateway instances. Ensures no single server is overwhelmed. Enables horizontal scaling under high load.
💡 Load Balancer = traffic director — spreads requests evenly so servers don't crash
NGINX AWS ALB Auto Scaling Horizontal scale
🔌
API Gateway
Receives encrypted behavioral streams from clients. Routes requests to appropriate microservices. Handles authentication, rate limiting, throttling, and API key management.
Node.js / Flask Rate limiting API keys Throttling
🗄️
Feature Store
Stores pre-computed behavioral features for each user — typing baseline, mouse velocity norms, device history. ML models read from here instead of recomputing each time.
💡 Feature Store = the ML system's "memory" of what normal looks like for each user
Feast Redis cache User baselines
🧠
ML Inference Service
Runs LSTM + Random Forest ensemble against incoming feature vectors. Scores each batch against user baseline from Feature Store. Outputs anomaly score 0–1 + confidence.
TensorFlow Serving ONNX Runtime <50ms inference
⚙️
Session Manager
Maintains live session state per user. Updates risk score in real time. Triggers the correct response action (allow / OTP / block) based on Decision Engine output.
Redis sessions WebSocket push
🔐 SECURITY LAYER
👤
Profile Store
Stores AES-256 encrypted behavioral profiles per user. Contains typing baseline, device fingerprints, trusted IPs, and historical session summaries. GDPR anonymized.
AES-256 MongoDB GDPR compliant
🔒 Zero Trust Architecture
"Never trust, always verify" — every single request is re-authenticated, regardless of source or previous session approval. No implicit trust granted to anyone.
› Every request verified with identity check
› Identity-based access control (RBAC)
› Behavioral score gates every action
📋
Audit Logger
Immutable, append-only log of all security events — every risk score, every session block, every OTP trigger. Used for forensic analysis and regulatory compliance.
💡 Immutable log = entries can never be deleted or altered — critical for audits
PostgreSQL Append-only WORM storage
🚨
Alert Engine
Fires alerts to admin when high-risk events occur. Sends push notifications, emails, and Slack alerts. Feeds anomaly data back into the retraining pipeline automatically.
PagerDuty Slack webhooks Email alerts
🔁 Model Lifecycle Management
🏋️
Training Pipeline
New labeled anomaly data → retrain LSTM + RF → generate updated model weights. Scheduled or triggered by alert volume spikes.
📌
Model Versioning
Each trained model tagged with version (e.g. v2.3.1). Old versions kept for rollback. MLflow / DVC tracks experiments.
⚗️
A/B Testing
New model served to 10% of traffic first. Metrics compared to current model. If better → full rollout. Eliminates risky big-bang deployments.
🔄
Retraining Loop
High-risk confirmed events automatically added to training dataset. Continuous learning ensures model adapts to new attack patterns.
☁️ Disaster Recovery & Infrastructure
💾
Backup Storage
All logs and user profiles backed up to AWS S3 / Azure Blob every 6 hours. Encrypted at rest with AES-256. Retention: 90 days.
🌍
Failover Region
Secondary deployment region (e.g. US-East → EU-West). If primary goes down, DNS auto-routes traffic to standby within 30 seconds. RTO < 1 min.
🔁
Data Replication
MongoDB replica sets + PostgreSQL streaming replication across 3 nodes. No single point of failure. Automatic leader election on node failure.
📊
Admin Dashboard
Real-time risk score visualization, live session map, anomaly heatmap, alert feed. Built on Grafana + Prometheus metrics pipeline.
🐳 Containerization & Testing
🐳
Docker — Service Containers
Each microservice (API Gateway, ML Inference, Session Manager, Alert Engine) runs in its own isolated Docker container. "Microservices deployed via containers."
☸️
Kubernetes — Orchestration
Manages container scaling, self-healing, rolling updates, and load distribution. Auto-scales ML pods under traffic spikes. HPA (Horizontal Pod Autoscaler) watches CPU/mem.
🧪
Unit & Integration Testing
Every service has unit tests (pytest / Jest). Integration tests verify cross-service data flow (e.g. Event Batcher → Kafka → ML Inference end-to-end).
🛡️
Load & Pen Testing
Load testing with k6 / Locust simulates 10k concurrent sessions. Penetration testing (OWASP) performed before every major release. SLA: 99.9% uptime.
Integration: State-Aware Auth layers on top of existing systems via SDK/API — no rip-and-replace of current infrastructure required. All components are independently deployable, horizontally scalable, and observable via Prometheus + Grafana.
