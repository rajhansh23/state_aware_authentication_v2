SLIDE 02 · SYSTEM INITIALIZATION

System Initialization:
How the Behavioral Baseline is Built

// brain-building phase · 5-step pipeline · no enforcement during learning
"This is the brain-building stage — if the baseline is strong, the entire security system becomes intelligent and reliable."
Phase 1
👤
New User
Registration
A new account is created. The system has no knowledge of how this user behaves yet — the slate is completely empty.
Account created · No baseline · No behavior history · Monitoring not yet active
Phase 2
👁
Passive Data
Collection
The system quietly watches and records how the user naturally interacts — typing, clicking, scrolling — over 7 to 14 days. No alerts are raised.
Silently observes behavior · 7–14 day window · User unaware · Zero disruption to experience
Phase 3
🧠
Profile
Construction
The ML model is trained on all collected data. It learns what "normal" looks like for this specific user. A behavioral baseline is set.
LSTM + RF trained · Behavioral clustering · Statistical mean & deviation calculated · Baseline locked
Phase 4
⚙️
Threshold
Calibration
The system fine-tunes its sensitivity. It decides how much change counts as "suspicious" — tuned to keep false alarms low without missing real threats.
Sensitivity tuned · False positive rate minimized · Risk thresholds set · Ready for enforcement
Phase 5
🛡
Active
Monitoring
The system is now fully live. Every session is silently checked against the baseline in real time. Any mismatch triggers the Decision Engine.
Live enforcement on · Real-time comparison · Session scoring active · Adaptive updates continue
⚡ Feature Engineering
Keystroke latency — exact time between each key press (in ms)
Mouse acceleration curve — how smoothly the user moves the cursor
Session entropy score — how random or predictable the user's actions are
Navigation sequence vectors — the order in which pages are visited
Time-of-day probability — when the user typically logs in
🤖 Baseline Model Details
LSTM sequence embeddings — the model stores behavioral patterns as a time-based memory map
Behavioral clustering (K-Means) — groups similar behaviors into "normal zones"
Statistical profile — mean & standard deviation for every tracked metric
"Baseline is not just stored data — it is a trained behavioral model."
🔄 Adaptive Learning
Updates baseline over time as user habits change
Handles behavior drift — new habits don't trigger false alerts
Seasonal patterns — adjusts for login time changes
🔒 Privacy Layer
On-device processing — data stays on user's device
Differential Privacy — mathematical noise added to protect identity
No raw data stored — only behavioral patterns are kept
