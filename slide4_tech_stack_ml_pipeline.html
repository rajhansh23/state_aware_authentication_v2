FEATURE ENGINEERING
Converts raw event data into meaningful numerical features that ML models can understand. Raw clicks become: typing speed (WPM), mouse velocity vectors, dwell time per key, swipe angle/pressure, scroll rhythm, etc. Feature engineering = translating human behavior into math.
48-dim feature vector Typing speed (WPM) Mouse trajectory Touch pressure Scroll rhythm

⚖️
NORMALIZATION LAYER
Standardizes all feature values using StandardScaler so different devices, screen sizes, and keyboard types don't bias the model. A fast typist on a desktop and a slow typist on mobile both produce comparable normalized values. Without this, models learn hardware differences, not behavior.
StandardScaler Z-score normalization Per-user baseline
📊 Device-agnostic
▼
normalized feature vector (48-dim) → ML inference
5
🧠
ML MODELS LAYER — PARALLEL INFERENCE
🔄
LSTM Network
Long Short-Term Memory — a type of recurrent neural network that has "memory". It reads behavior sequences over time (t-3 → now) and detects gradual drifts invisible in a single snapshot.
→ Detects unusual sequences & behavioral drift
🌲
Random Forest
Ensemble of Decision Trees — hundreds of trees each "vote" on whether behavior is normal or anomalous. Uses static feature snapshots. Extremely fast and handles imbalanced data well.
→ Fast classification of known attack patterns
🏆
Ensemble Scorer
Weighted Model Combination — fuses LSTM's temporal score + Random Forest's classification score using learned weights. Final output is a single anomaly score (0→1) representing threat level.
→ Boosts accuracy to ~95%, reduces false positives
TensorFlow / PyTorch scikit-learn ONNX cross-platform Python 3.x
▼
anomaly score [0.00–1.00] + confidence + context → decision engine

⚙️
DECISION ENGINE
Takes the ML anomaly score and combines it with rule-based logic (IP reputation, time-of-day, login history) to assign a final Risk Level. ML gives the score — the decision engine interprets it and decides the action. Without this layer, ML only outputs a number and takes no action.
Rules Engine Node.js / Flask API Context enrichment
🎯 Smart risk decision

📚 Key Terms Explained
🔄 LSTM
Long Short-Term Memory — an AI model that "remembers" past behavior and detects patterns evolving over time.
🌲 Random Forest
Hundreds of decision trees each vote on a classification; majority wins. Fast and accurate for known threat patterns.
⚡ Apache Kafka
A message queue system that handles millions of events per second reliably — the "highway" for data flow.
🔧 Feature Engineering
Transforming raw data (e.g. click timestamps) into numbers that represent meaning (e.g. typing speed in WPM).
⚖️ Normalization
Rescaling values to a standard range so a fast typist and slow typist are fairly compared by the model.
🏆 Ensemble Model
Combines outputs of multiple models with weighted voting — more reliable than any single model alone.
🔁 Continuous Learning
The system retrains itself on new data periodically, adapting to new attack patterns and user changes.
🔒 Zero Trust Architecture
Security principle: "Never trust, always verify." Every request is authenticated regardless of source.
